# 01 — Project Brief (PRD)

## 1. Objective
Build a **single-page** personal portfolio for **Muddassir Ali** that reads like it was hand-built by a senior developer/marketer — not generated by a template or AI. Primary job: convert a **recruiter / HR / hiring manager** skim (15–40 seconds) into "let's interview this person."

## 2. Primary audience
- In-house talent teams & recruiters hiring for **Performance Marketing / Meta Ads / Growth / AI Automation** roles.
- Secondary: agency founders & potential retainer clients.

Because the audience is hiring-focused, the site is **credibility-first**: proof, scope, systems built, tools mastered. No hype, no "guru" tone.

## 3. Positioning (the one line)
> **Meta Ads & Performance Strategist who engineers the automation behind the ads.**
Sub-identity: *Practitioner — I build the systems, not just run them.*

## 4. Non-negotiable constraints
- **Single page.** All sections scroll on one route (`/`). Anchor nav only.
- **White / light theme.** Bright, clean, editorial. NOT dark.
- **"Real developer" aesthetic.** Sharp, precise, restrained. See Design System for the anti-AI-fake rules.
- **Client details hidden.** No client names, domains, or third-party team names anywhere. SEO work shown by *industry + scope* only.
- **No fabricated metrics.** Placeholders where real numbers aren't confirmed.
- **Fast.** Lighthouse performance ≥ 90 mobile. Motion must never block reading.

## 5. Tech stack (recommended)
- **Framework:** Next.js 14 (App Router) + TypeScript
- **Styling:** Tailwind CSS (custom tokens from Design System)
- **Motion:** Framer Motion (component/enter animations) + GSAP ScrollTrigger (scroll-linked) + Lenis (smooth scroll)
- **Fonts:** `Geist` (sans) + `Geist Mono` (mono) via `next/font` — self-hosted, no layout shift
- **Icons:** `lucide-react` (thin, consistent). No emoji in UI.
- **Deploy:** Vercel. Domain: muddassirali.com

*If Antigravity prefers a Vite + React single-page setup, that's acceptable — keep the same tokens, motion libraries, and section structure.*

## 6. Page map (single scroll)
1. Sticky nav
2. Hero
3. Tool/skill marquee (trust bar)
4. About + principles
5. Selected Work (own products/builds)
6. Case Studies (anonymized SEO/performance)
7. Capabilities / Stack
8. Experience timeline
9. Impact metrics strip (animated counters)
10. Contact / CTA
11. Footer

## 7. Success criteria (definition of done)
- [ ] Loads clean on mobile + desktop, no CLS jank
- [ ] Every section animates in *once* on scroll, subtly
- [ ] Resume downloads from nav + hero + contact
- [ ] All links work (LinkedIn, YouTube @MuddassirAli, email, phone)
- [ ] Zero client-identifying info visible or in page source
- [ ] Lighthouse: Perf ≥90, A11y ≥95, Best Practices ≥95, SEO ≥95
- [ ] `prefers-reduced-motion` respected — animations reduce to fades
- [ ] View source looks like a real dev wrote it: semantic HTML, sensible component names

## 8. Guardrails for the AI builder (learned the hard way)
- Build **in phases**, one section per step. Never rewrite the whole file in one destructive pass.
- After each section, stop and let it render before continuing.
- Keep all copy from the `content/` files verbatim unless asked to trim.
- Do not add stock hero images of "marketing dashboards" — this is a typography-led site.
